このブログは 2020/12 から Gatsby.js を使ってたらしいんだけど、実装はほとんど放置していて新しい環境で clone してもなんかうまく動かないことが多くて、Gatsby.js も何一つ覚えてないしこの際だから書き直そうということで Next.js を使う。

記事は普通の Markdown で書いていたんですが、以前は Youtube の埋め込みをたぶん Gatsby.js のプラグインか何かでやっていて、そのあたりを MDX とかいうフォーマットを使うことで解決しようとしている。

なんか React? の Compornent? を埋め込めるらしいぞ。よく知らんが。

Next.js を選んだのはWebアプリとかにも応用できそうだから、Gatsby.js だと SSG の域を出れなさそうな気がして。これも良くは知らんが。

Gatsby.js と Next.js どっちも React ベースらしいけど、Gatsby.js はちょっと独特な感じがしている。純粋に React を扱うというか学ぶんだったら Next.js の方がいいかなーと。Gatsby.js は GraphQL で行きやすいっていうのもまあ学び的には良いとは思うけど、あまり技術スタックを広げ過ぎると浅瀬過ぎてしまって後で見たときにわからなさすぎること考えている。もともとフロントエンド側の人間じゃないのでなるべく絞りたいという感じ。

Next.js(TypeScript)/TailwindCSS + Vercel でやっていこうと思ってる。今のところ。

とりあえず苦労しながら完成までは出来たけど (デザインはほぼ変わらない)、React の R の字も分かってない感が否めない。この後は Podcast サイトや己のホームページの一部を Next.js へ移そうと思うので、まあちょっとずつ学びがあればいいやという感じ。

出来たとばかり思ってたけどツイートボタンちゃんと出来てないわ。今直しています。
